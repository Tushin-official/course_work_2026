# 1) market_graph.py

Задача: построить веса рёбер w[i,j] для направленного графа.

Узлы: 0 - dummy, 1..N - акции.

Вес ребра (i -> j) для i,j != 0:

w[i,j] = S[i-1, j-1] * (ask[j-1] - bid[i-1])

Для dummy рёбра 0->k и k->0 имеют вес 0 (чтобы цикл через dummy просто выбирал "endpoints", а стоимость давал только путь между акциями).

Вход: ask(N), bid(N), sim(N×N)
Выход: w((N+1)×(N+1))

# 2) qubo_cycle.py

Задача: собрать QUBO-формулировку - найти минимальный по весу цикл (с dummy)

Бинарные переменные b_{i,j} ∈ {0,1} для всех направленных рёбер i->j, где i != j, и i,j ∈ {0..N}.

1. Создаёт VarMap: маппинг (i,j) <-> u (индекс переменной QUBO).

2. Собирает:

    a) Линейную часть lin[u] (стоимость ребра)

    b) Квадратичную часть quad[(u,v)] (пенальти/ограничения + tabu)

Какие ограничения туда зашиты?

Выбранные b_{i,j} должны описывать один направленный цикл (и мы потом проверяем, что он содержит dummy).

Пенальти обеспечивают:

1) out(i) <= 1 и in(i) >= 1 (не больше 1 входа/выхода у узла)

2) Баланс потока out(i) = in(i) (если у узла есть ребро наружу, то есть и ребро внутрь)

3) Запрет выбрать сразу (i->j) и (j->i) одновременно (антипараллель)

4) tabu: штраф на конкретную пару endpoints через dummy (b_{0,short} * b_{long,0})

Вход: w, tabu (опционально), mp (сила штрафа), mc
Выход: (lin, quad, var_map)

# 3) ising_mapping.py

Задача: перевести QUBO в Ising так, чтобы SB мог минимизировать энергию.

1) QUBO переменные: z_u ∈ {0,1}

2) Ising спины: s_u ∈ {-1,+1}

Связь: z = (s+1)/2

Функция qubo_to_ising(lin, quad) возвращает:

1) J - матрицу связей Ising (размера M×M)

2) h - поле (вектор M×1)

const - константу энергии (для оптимизации не важна)

Вход: lin, quad
Выход: J, h, const

# 4) pair_search.py

Задача: "обвязка" вокруг SB - запускаем много раз, декодируем, проверяем, извлекаем пару, ведём tabu, собираем несколько пар.

Главные части:

1. solve_best_cycle_with_sb(...)

    Строит QUBO -> Ising (J,h). Делает n_runs запусков SB (разные сиды)

    Каждый запуск:

        a) SB даёт финальные x

        b) Переводим в спины: s = sign(x) → в биты z = (s+1)/2

        c) По z восстанавливаем список выбранных рёбер

        d) Проверяем валидность и стоимость

        e) Возвращает лучшее валидное решение (минимальный path_weight).

2. verify_single_dummy_cycle(...)

    Это "фильтр" от плохих конфигураций: проверяет, что:

        a) Степени по in/out корректны

        b) Реально есть цикл

        c) Цикл включает dummy и только один

        d) Нет split cycles / циклов без dummy

3. extract_pair_and_weight(...)

    Из цикла [0, short, ..., long, 0] делает:

        a) short = cycle[1]

        b) long = cycle[-2]

        c) Путь path_nodes = cycle[1:-1]

        d) path_weight = sum(w[a,b] по ребрам пути)

4. TabuList

    Хранит матрицу tabu и запрещает повторять одну и ту же пару endpoints.

5. find_multiple_pairs(...)

    Повторяет поиск лучшего цикла. Если path_weight < threshold, принимает пару, добавляет tabu на найденную пару, ищет следующую (до max_pairs)

Вход: w, threshold, параметры SB
Выход: список CycleSolution (или (solution,traj) если ты добавлял возврат траекторий)

# 6) dtw_similarity.py

Задача: из временных рядов цен построить матрицу похожести S.

1. build_daily_sequences_from_hourly_klines(klines_1h):

    a) Группирует 1h свечи по UTC-дням

    b) Нормирует внутридневную последовательность на base_open дня

    c) Возвращает {day: sequence}

2. compute_similarity_matrix(symbols, daily_seq, ...):

    a) Выбирает общие дни у всех символов

    b) Берёт последние 5 полных дней (обычно исключая текущий)

    c) Для каждой пары считает DTW-distance (среднее по дням)

    d) Нормирует distance -> similarity S ∈ [0,1] (min-max inversion)

Вход: свечи
Выход: S и список дней, по которым считали

# 8) verify_bruteforce.py

Задача: строгая проверка качества на маленьком N:

Для каждого (short,long) считает точный минимум по всем простым путям (динамика по маскам, O(N^2 * 2^N)).

Сравнивает SB-решение с exact:

diff проверяет корректность подсчёта веса пути (у тебя diff=0 — отлично)

gap показывает, насколько SB не доходит до оптимума
